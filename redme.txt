 الخطوات الاحترافية التالية المخصصة لبيئة العمل على جهازك في مجلد البوت F:\tota_story_bot:
1️⃣ الخطوة الأولى: منع جهاز الكمبيوتر من النوم (Sleep Mode)
لكي يستمر البوت في استقبال وإرسال الرسائل، يجب ألا يدخل نظام التشغيل في وضع السكون:
في نظام Windows:
افتح الإعدادات (Settings) 🡨 النظام (System) 🡨 الطاقة والبطارية (Power & battery).
افتح خيار الشاشة وحالة السكون (Screen and sleep).
قم بتغيير خيار "عند التوصيل بالكهرباء، ضع الجهاز في وضع السكون بعد..." إلى أبداً (Never).
2️⃣ الخطوة الثانية: استخدام مدير العمليات الاحترافي PM2 (الخيار الأفضل والموصى به للـ Node.js)
إذا قمت بتشغيل البوت بالأمر العادي node bot.js وحدث أي انقطاع مفاجئ في الإنترنت أو خطأ برمجلي مؤقت، فسيتوقف البوت تماماً عن العمل.
أداة PM2 هي مدير عمليات ذكي يقوم بإعادة تشغيل البوت تلقائياً ومباشرة في الخلفية في حال حدوث أي توقف أو خطأ.
طريقة التثبيت والتشغيل باستخدام PM2:
افتح موجه الأوامر (CMD/Terminal) كمسؤول (Run as administrator).
قم بتثبيت أداة PM2 على جهازك بالكامل عبر كتابة الأمر التالي:
code
Bash
npm install -g pm2
انتقل إلى مجلد البوت الخاص بك:
code
Bash
F:
cd F:\tota_story_bot
قم ببدء تشغيل البوت باستخدام PM2 وتسميته باسم مميز (مثلاً tota-story-bot):
code
Bash
pm2 start bot.js --name "tota-story-bot"
أوامر مفيدة لإدارة البوت باستخدام PM2:
لمراقبة حالة البوت والتأكد من أنه يعمل: pm2 status
لمشاهدة السجلات والرسائل الواردة مباشرة (Logs): pm2 logs
لإيقاف البوت: pm2 stop tota-story-bot
لإعادة تشغيل البوت يدوياً: pm2 restart tota-story-bot
3️⃣ الخطوة الثالثة: تشغيل البوت تلقائياً عند إقلاع ويندوز (Windows Startup) (اختياري)
إذا أردت أن يعمل البوت تلقائياً فور تشغيل جهاز الكمبيوتر أو بعد انقطاع الكهرباء وإعادة التشغيل، يمكنك عمل ملف تشغيل تلقائي كالتالي:
اضغط على زري Windows + R على لوحة المفاتيح لفتح نافذة التشغيل واكتب: shell:startup ثم اضغط Enter. سيفتح لك هذا مجلد "بدء التشغيل التلقائي برامج الويندوز".
بداخل هذا المجلد، انقر بزر الفأرة الأيمن واختر جديد (New) 🡨 مستند نصي (Text Document).
افتح الملف النصي واكتب بداخله الأسطر التالية (مع تعديل مسار المجلد إن تطلب الأمر):
code
Bat
@echo off
cd /d F:\tota_story_bot
pm2 start bot.js --name "tota-story-bot"
احفظ الملف بإسم: run_bot.bat (تأكد من تغيير صيغة الملف من .txt إلى .bat).
الآن، في كل مرة يشتغل فيها جهاز الكمبيوتر الخاص بك، سيقوم النظام تلقائياً بتشغيل البوت وإبقائه حياً في الخلفية وبشكل متزامن بالكامل مع لوحة التحكم السحابية دون أي تعارضات!

-----------------

1️⃣ حل مشكلة التعارض وإيقاف التداخل (409 Conflict)
سبب المشكلة:
تليجرام يمنع نهائياً تشغيل نسختين من نفس البوت (باستخدام نفس التوكن) في نفس الوقت. من سجلاتك نلاحظ التالي:
تشغيل البوت في الخلفية عبر PM2 (باسم tota-story-bot وحالته online).
وفي نفس الوقت، قمت بتشغيل الأمر node bot.js يدوياً في الطرفية (Terminal).
هذا التزامن جعل النسخة المفعّلة في الخلفية عبر PM2 والنسخة اليدوية في سطر الأوامر تتنافسان على استقبال التحديثات من خوادم تليجرام، مما سبّب الخطأ 409 Conflict.
💡 الحل الصحيح والآمن:
بما أنك قمت بإعداده مسبقاً بنجاح ليعمل دائماً في الخلفية عبر PM2، فلا تحتاج إلى تشغيل الأمر node bot.js يدوياً.
لإدارة البوت ومراقبة أدائه بكفاءة أثناء تشغيله في الخلفية، استخدم الأوامر التالية من سطر الأوامر:
لمشاهدة سجلات البوت الحية (الـ Logs):
code
Bash
pm2 logs 0
أو:
code
Bash
pm2 logs tota-story-bot
(هذا سيعرض لك عمل البوت أولاً بأول كما لو أنك شغلت node bot.js ولكن دون إحداث تداخل أو تعارض).
لإعادة تشغيل البوت عند تحديث الأكواد:
code
Bash
pm2 restart tota-story-bot
لإيقاف أداة PM2 بالكامل (إذا كنت ترغب بالتشغيل اليدوي بدلاً منها مؤقتاً):
code
Bash
pm2 stop tota-story-bot
2️⃣ حل مشكلة انقطاع الاتصال بـ Firestore والشبكة المحلية
سبب المشكلة:
يواجه جهاز الكمبيوتر لديك صعوبة في حل عناوين خوادم جوجل السحابية عبر بروتوكول gRPC الافتراضي لشاشات الحماية أو مشاكل في DNS الشبكة الخاصة بك (Name resolution failed for target dns:firestore.googleapis.com). كذلك، يظهر لديك تحذير بأن ملف التكوين firebase-applet-config.json غير موجود في المجلد المحلي لجهازك مما يجعله يتصل بقاعدة بيانات تجريبية افتراضية قد تختلف عن السحابة الفعلية المنشأة لمشروعك.
💡 ما قمت بإصلاحه وتجهيزه لك الآن:
تحديث ملف bot.js السحابي: قمت بتعديل كود البوت لتثبيت بروتوكول الاتصال السحابي الآمن Long Polling (بتفعيل experimentalForceLongPolling: true). هذا التحديث يضمن تخطي جدران حماية الشبكة المنزلية أو مشكلات DNS الخاصة بهيئات الاتصال المحلية، مما يوفر اتصالاً فائق الثبات ومضمون لـ Firestore بنفس أسلوب الأمان المعتمد في تطبيق الويب.
اتخاذ إجراء من طرفك لتحديث الأكواد على حاسوبك:
قم بنسخ ملف bot.js الجديد ومحتوياته من السحابة إلى مجلد البوت على جهازك F:\tota_story_bot\bot.js لتضمن تشغيل التقنية الجديدة والمطوَّرة.
تنزيل ملف الإعدادات: تأكد من تحميل ملف التكوين الخاص بقاعدة بياناتك السحابية والمسمى firebase-applet-config.json من واجهة محرر الأكواد السحابية هنا ووضعه مباشرة داخل مجلد البوت على حاسوبك الشخصي بجانب ملف bot.js. وجود هذا الملف ضروري ليعرف البوت كيف يتوجه تلقائياً لقاعدة بياناتك المخصصة لمشروعك دون حدوث أي تضارب.
🚀 ملخص خطوات التشغيل النهائي على الحاسوب:
انسخ ملف bot.js المحدَّث وضعه في مجلد البوت المحلي بجهازك.
ضع ملف firebase-applet-config.json في نفس المجلد.
افتح سطر الأوامر في مجلد البوت واكتب ما يلي لإلغاء التعارض وتحديث الأداة في الخلفية:
code
Bash
pm2 stop tota-story-bot && pm2 delete tota-story-bot
pm2 start bot.js --name "tota-story-bot"
pm2 save
راقب عمل البوت بثبات وسلاسة من خلال سطر الأوامر:
code
Bash
pm2 logs tota-story-bot
سيعمل البوت الآن على حاسوبك 24/7 بمنتهى الثبات وبمزامنة حقيقية وفورية مع قاعدة البيانات السحابية الآمنة! لا تتردد في طرح أي استفسار آخر لمساعدتك.
